### 세그먼트별 상세 평가

#### A. Claude Code 초보자 (글로벌) — 15점

**고통의 강도: 5/5**
- "뭘 깔아야 할지 모르겠다"는 가장 빈번한 질문 (Reddit 다수)
- "반나절 시행착오" 낭비 (일상적 좌절)
- 9,000+개 앞에서 선택의 마비
- 증거: "I wish there had been a resource like this back then" (score 3,238)

**지불 의향: 2/5**
- 현재 무료 서비스로 충분히 만족할 가능성 높음
- 아직 Claude Code 자체에 돈을 쓰는 단계 (Pro $20~Max $100)
- 추가 도구에 지불 동기 낮음

**시장 점유 가능성: 4/5**
- 직접 경쟁자 적음 (Anthropic Setup 플러그인만 유사)
- 프리셋 팩 기능으로 명확한 차별화
- 단, 글로벌 규모라 "60% 점유"는 어려움

**추천/확산력: 4/5**
- r/ClaudeAI (42만+ 구독자) 같은 활발한 커뮤니티 존재
- 초보자는 "이거 좋았다"를 적극 공유하는 경향
- 단, 글로벌이라 입소문 분산

---

#### B. 중급 실험가 (글로벌) — 13점

**고통의 강도: 4/5**
- MCP 축적, 토큰 낭비, 응답 품질 저하
- "plugin fatigue" 현상 관측
- 증거: "my .claude.json accumulated servers I'd forgotten about"

**지불 의향: 3/5**
- 이미 Claude Code에 상당 비용 투자 ($20~$200/월)
- 토큰 절약 = 직접적 비용 절감 → 지불 동기 존재
- 단, 무료 대안(MCP-tidy 등)과 비교

**시장 점유 가능성: 3/5**
- 조합 분석/점수화는 유일하지만, 세그먼트 규모가 초보자보다 작음
- 파워유저일수록 자체 해결 선호

**추천/확산력: 3/5**
- 기술 블로그, Dev.to 등에서 공유 가능
- 단, 추천보다 "내가 만든 도구" 공유 문화가 더 강함

---

#### C. 팀 리더/DevOps — 15점

**고통의 강도: 4/5**
- 팀원 불일치, 보안 우려, 반복 온보딩
- 증거: IMWEB DevOps팀 10개+ MCP 도입 시행착오

**지불 의향: 5/5**
- 팀 생산성에 직결 → 가장 명확한 ROI
- 기업 예산으로 결제 가능
- 1:1 세팅 점검, 팀 온보딩 문서 = 유료 서비스 자연스러운 연결

**시장 점유 가능성: 2/5**
- B2B 세일즈 사이클이 길고, 현재 세일즈 팀/프로세스 없음
- 각 기업의 요구사항이 커스텀화 필요
- 시장 규모 자체가 작음 (아직 팀 단위 도입 초기)

**추천/확산력: 4/5**
- 팀 내 공유 → 자연스러운 확산
- 기업 사례 = 강력한 레퍼런스
- 단, 초기 1-2개 레퍼런스 확보가 어려움

---

#### D. 한국 Claude Code 사용자 — 16점 (최고)

**고통의 강도: 4/5**
- 영어 문서 장벽 + Windows 환경 문제
- 한국어 플러그인 추천 자료 전무
- 증거: 한국 블로그/커뮤니티에서 MCP 설정 가이드 수요

**지불 의향: 2/5**
- 무료 도구로 시작, 유료 전환은 나중
- 한국 개발자 커뮨니티의 무료 도구 선호 경향
- 단, 팀/기업 도입 시 지불 가능성

**시장 점유 가능성: 5/5**
- **한국어 Claude Code 플러그인 추천 도구 = 현재 0개** → 독점 가능
- 한국 시장은 글로벌 대비 작지만, 집중하면 빠르게 60%+ 점유 가능
- 이미 한국어 UI + 6개 한국어 가이드 보유 (경쟁 해자)

**추천/확산력: 5/5**
- 한국 개발자 커뮤니티 밀집도 높음 (GeekNews, Velog, 카카오/네이버 커뮤니티)
- "한국어 도구" → 애국심/자부심 기반 공유 동기
- PyTorch 한국 커뮤니티, 클라우드/AI 밋업 등 접근 가능
- 한국 → 일본/동남아 등 비영어권 확장 레퍼런스

---

#### E. ChatGPT 이주자 — 14점

**고통의 강도: 5/5**
- 생태계 전환 시 모든 것이 새로움
- "Most of us don't fully understand tokens yet" (score 1,805)

**지불 의향: 2/5**
- Claude 구독료 자체가 새로운 지출, 추가 도구에 지불 의향 낮음

**시장 점유 가능성: 3/5**
- 대규모 유입이지만, "이주자"라는 세그먼트는 일시적 (정착 후 초보자/중급으로 전환)

**추천/확산력: 4/5**
- "Claude로 옮겼는데 이거 좋다" 공유 문화 활발
- 단, 세그먼트 정체성이 일시적

## Step 6: 비치헤드 선정

### 1차 비치헤드: D. 한국 Claude Code 사용자

**선정 이유:**
- **시장 독점 가능**: 한국어 플러그인 추천 도구 = 현재 0개. 진입하면 바로 100% 점유
- **확산력 최고**: 밀집된 커뮤니티, "한국어 도구" 공유 동기 강함
- **이미 준비됨**: 한국어 UI, 6개 한국어 가이드, Windows 문제 해결 가이드 보유
- **실행 가능**: 글로벌 경쟁 없이, 한국 커뮤니티 집중으로 빠른 PMF 검증
- **확장 경로**: 한국 → 글로벌 (영어 지원 이미 있음), 개인 → 팀/기업

**리스크:**
- 시장 규모가 글로벌 대비 작음 → 성장 천장
- 지불 의향이 낮음 → 수익화는 2차 세그먼트(팀 리더)에서

### 2차 확장 세그먼트: A. Claude Code 초보자 (글로벌)

**확장 시점:** 한국 시장에서 PMF 검증 + 레퍼런스 확보 후
**확장 방법:** 영어 콘텐츠 마케팅 (Reddit, Dev.to, YouTube)

### 3차 수익화 세그먼트: C. 팀 리더/DevOps

**확장 시점:** 개인 사용자 기반 확보 후
**확장 방법:** 유료 서비스 (1:1 세팅 점검, 팀 온보딩 문서)
