# North Star Metric: Plugin Advisor

*작성일: 2026-03-29*

---

## Step 1: Business Game 분류

**Productivity Game**

Plugin Advisor는 사용자가 Claude Code 플러그인 세팅을 더 **효율적으로, 더 안전하게** 완료하도록 돕는 도구다. 시간 절약 + 성공적 세팅이 핵심 가치.

- Attention Game 아님 — 오래 머무르는 것이 가치가 아님 (빠를수록 좋음)
- Transaction Game 아님 — 거래가 아닌 도구 사용
- **Productivity Game** — 효율적으로 목표(세팅 완료)를 달성

### 선정: 월간 추천 완료 수 (Monthly Recommendations Completed)

> **"한 달 동안 사용자가 플러그인 추천/조합 분석을 완료한 횟수"**

**정의:** Advisor에서 추천 결과를 받거나, Optimizer에서 조합 분석 결과를 본 고유 세션 수 (월간)

**7가지 기준 검증:**

| 기준 | 충족 여부 | 설명 |
|------|----------|------|
| 이해 용이 | O | "이번 달에 몇 번의 추천/분석이 완료됐는가" |
| 고객 중심 | O | 추천 완료 = 사용자가 실제 가치를 받은 순간 |
| 지속 가치 | O | 새 프로젝트마다, 플러그인 변경 시마다 재사용 |
| 비전 정렬 | O | "플러그인 세팅 = Plugin Advisor" 비전과 직결 |
| 정량적 | O | 이벤트로 정확히 측정 가능 |
| 실행 가능 | O | UX 개선, 콘텐츠, SEO 모두 이 지표에 영향 |
| 선행 지표 | O | 추천 완료 ↑ → 설치 스크립트 복사 ↑ → 유료 서비스 전환 ↑ |


### Input 1: 월간 신규 방문자 수
- **정의:** 사이트에 처음 방문한 고유 사용자 수
- **영향:** 퍼널 상단. 콘텐츠 마케팅, SEO, 커뮤니티 홍보로 개선
- **목표 (90일):** 500+/월
- **측정:** Vercel Analytics

### Input 2: 분석 시작률 (Visit → Start)
- **정의:** 방문자 중 Advisor 또는 Optimizer에서 입력을 시작한 비율
- **영향:** 랜딩 페이지 UX, CTA 명확성, 온보딩 흐름
- **목표:** 30%+
- **측정:** 이벤트 추적 (advisor_start, optimizer_start)

### Input 3: 분석 완료률 (Start → Complete)
- **정의:** 분석을 시작한 사용자 중 결과 화면까지 도달한 비율
- **영향:** 입력 UX, 로딩 속도, 에러 핸들링
- **목표:** 70%+
- **측정:** 이벤트 추적 (advisor_complete, optimizer_complete)

### Input 4: 재방문율
- **정의:** 30일 내 2회 이상 방문한 사용자 비율
- **영향:** 콘텐츠 가치, 이메일 리텐션, 새 기능/가이드 추가
- **목표:** 20%+
- **측정:** Vercel Analytics

### Input 5: 공유 유입 방문자
- **정의:** 공유 링크/SNS를 통해 유입된 방문자 수
- **영향:** Growth Loop (Viral Loop) 건강도, 공유 기능 UX
- **목표:** 50+/월
- **측정:** UTM 파라미터 + referrer 추적

## 즉시 실행 필요 사항

현재 서버 측 Analytics가 없으므로, 이 지표 체계를 측정하려면:

1. **Vercel Analytics 활성화** — 방문자, 페이지뷰, referrer
2. **이벤트 추적 서버 전송** — 현재 localStorage만 사용 중 → 서버로 전송 추가
   - `advisor_start`, `advisor_complete`
   - `optimizer_start`, `optimizer_complete`
   - `script_copy`, `share_click`
3. **UTM 파라미터 지원** — 채널별 유입 추적
