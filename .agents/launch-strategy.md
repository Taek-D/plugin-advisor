# Launch Strategy: Plugin Advisor

*작성일: 2026-03-29*
*런칭 타겟: 한국 Claude Code 사용자 (비치헤드)*
*기반: gtm-strategy.md, product-marketing-context.md*

---

## 런칭 유형

**무엇을:** 이미 배포된 제품의 "공식 소개" (v1.3, 51개 DB, Optimizer 기능 포함)
**현재 상태:** 배포 완료 (plugin-advisor.vercel.app), 공식 홍보 미시작
**현재 채널:** 없음 (이메일 리스트, 블로그, 커뮤니티 모두 없음)

→ Phase 1(Internal)~3(Beta)을 건너뛰고 **Phase 4(Early Access) + Phase 5(Full Launch)를 동시 진행**하는 전략

## ORB 채널 전략

### Owned (소유 채널) — 장기 자산

| 채널 | 현재 상태 | 런칭 전 준비 | 우선순위 |
|------|----------|-------------|---------|
| **사이트 블로그/가이드** | 6개 가이드 존재 (/guides) | SEO 메타 최적화, 가이드 3편 추가 | 1순위 |
| **이메일 리스트** | 없음 | 사이트에 뉴스레터 구독 폼 추가 | 2순위 |
| **GitHub 리포** | private | 공개 전환 검토 (오픈소스 신뢰도) | 3순위 |

**핵심 원칙:** 모든 Rented/Borrowed 채널의 트래픽은 Owned 채널(사이트)로 유도

### Borrowed (빌린 채널) — 신뢰도 부스트

| 채널 | 액션 | 시점 |
|------|------|------|
| **한국 AI 뉴스레터** | 소개 요청/기고 | Launch +1주 |

## 주의사항

1. **"Show, don't tell"** — 기능 설명보다 데모 GIF가 10배 효과적
2. **댓글 = 금** — 런칭 후 48시간 모든 댓글에 빠르게 응답
3. **피드백 → 즉시 반영** — "이 기능 추가해주세요" → 빠르게 반영하면 충성 사용자 확보
4. **과대 광고 금지** — 브랜드 톤("실용적, 솔직한, 사용자 친화적")과 일관되게
5. **한 번에 다 하지 않기** — GeekNews 먼저, 반응 보고, 다음 채널로
