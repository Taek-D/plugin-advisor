**2. Create `components/RecommendPluginCard.tsx`** — Shared card for complement suggestions:

a) Props: `{ pluginId: string; reason: string }`.

b) Look up plugin data: `const plugin = PLUGINS[pluginId]`. Guard: if not found, return null.

c) Card layout:
- Left: `getCategoryIcon(plugin.category)` icon
- Center: Plugin name as `<Link href={"/plugins/" + plugin.id}>` (blue/accent underline on hover), one-line `plugin.desc`, reason text in muted color
- Right: Copy install button

d) Copy install button: `navigator.clipboard.writeText(plugin.install.join("\n"))`. Show `t.optimizer.installCopy` normally, `t.optimizer.installCopied` for 1.5s after click. Use `Copy` / `Check` icons from lucide-react.

e) Styling: `surface-panel-soft rounded-2xl p-4` with flex layout.

**3. Create `components/ReplacementCard.tsx`** — Card for replacement suggestions:

a) Props: `{ original: string; reason: "unverified" | "partial" | "deprecated"; replacement: string | null }`.

b) Look up both plugins: `PLUGINS[original]` and `PLUGINS[replacement]` (replacement may be null).

c) Card layout:
- "Original" plugin name (strikethrough or dimmed) with ArrowRight icon pointing to "Replacement" plugin name (as Link to /plugins/[id])
- Reason badge: map reason to localized text — "unverified" -> "미검증"/"Unverified", "partial" -> "부분검증"/"Partial", "deprecated" -> "지원중단"/"Deprecated"
- If replacement exists: show replacement plugin desc + copy install button (same pattern as RecommendPluginCard)
- If replacement is null: show "대안 없음" / "No alternative available" message

d) Styling: `surface-panel-soft rounded-2xl p-4`.

**4. Create `components/ComplementSection.tsx`** — Collapsible complement recommendations:

a) Props:
```typescript
type ComplementSectionProps = {
  complements: ComplementSuggestion[];
  open: boolean;
  onToggle: () => void;
};
```
The `open` state and `onToggle` are controlled by the parent (ResultsPanel) so CoverageGrid's scroll-to-open can work.

b) Section header: clickable button with `t.optimizer.complementTitle` + count badge + ChevronDown icon that rotates 180deg when open. Use `aria-expanded={open}`.

c) Collapsible body: `overflow-hidden transition-all duration-300 ease-in-out` with `style={{ maxHeight: open ? "9999px" : "0px" }}`.

d) Inside: map `complements` to `<RecommendPluginCard>` entries. The reason for each: use `t.categories[complement.forCategory]` to say which uncovered category this fills (e.g., format as the category name the complement covers).

e) If `complements.length === 0`: show a positive "All categories covered" message instead of the card list.

f) Wrap outer container in Card with `surface-panel rounded-[28px] p-5`.

**5. Create `components/ReplacementSection.tsx`** — Collapsible replacement recommendations:

a) Props: `{ replacements: ReplacementSuggestion[] }`.

b) Internal `const [open, setOpen] = useState(false)` — this section manages its own open state (no external scroll-to needed).

c) Same collapsible pattern as ComplementSection: header button with `t.optimizer.replacementTitle` + count + ChevronDown, `aria-expanded`, max-height transition.

d) Inside: map `replacements` to `<ReplacementCard>` entries.

e) If `replacements.length === 0`: do NOT render the section at all (no replacements = no section needed, unlike complements where coverage feedback is always useful).

f) Wrap in Card with `surface-panel rounded-[28px] p-5`.

**Anti-patterns to avoid:**

- Do NOT call `scorePlugins` — it was already called in OptimizerApp; just consume the result props
- Do NOT access `plugin.conflicts[]` directly — use the conflicts from ScoringResult
- Plugin install commands are `string[]` — always `.join("\n")` for copy
