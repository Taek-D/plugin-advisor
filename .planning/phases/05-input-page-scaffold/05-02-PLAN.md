2. **Create `components/OptimizerApp.tsx`** — 'use client' root component. This is the state machine for the optimizer input page.

State:
- `selectedPlugins: Plugin[]` — plugins chosen by user (from paste or manual)
- `unmatched: string[]` — tokens from paste that didn't resolve
- `activeTab: "paste" | "type"` — current input mode

Key handlers:
- `handlePasteResult(result: ParseResult)` — receives parsed paste result, looks up matched ids in PLUGINS object, adds to selectedPlugins (avoiding duplicates), sets unmatched
- `handleAddPlugin(pluginId: string)` — adds plugin from autocomplete to selectedPlugins (avoid duplicates)
- `handleRemovePlugin(pluginId: string)` — removes from selectedPlugins
- `handleSampleData()` — inserts a sample `claude mcp list` output (3-4 realistic plugins in Variant A format: "context7 (user):\nplaywright (user):\ngithub (user):"), parses it, and populates selectedPlugins

Layout structure (follow existing advisor page visual style with Card wrapping):
```
Header section:
  - Page title (t.optimizer.pageTitle) + subtitle (t.optimizer.pageSubtitle)
  - AI Coming Soon button (disabled, aria-disabled="true", opacity-60 cursor-not-allowed)
    Use Sparkles icon from lucide-react. Text: t.optimizer.aiComingSoon
    Do NOT use pointer-events-none (breaks title tooltip)

Input section (Card):
  - TabsList with two TabsTriggers: tabPaste / tabType
  - Conditionally render McpPasteInput or PluginTypeInput based on activeTab

Selected plugins section:
  - If selectedPlugins.length > 0: render SelectedPluginChips
  - If unmatched.length > 0: render unmatched section with heading t.optimizer.unrecognizedPlugins and each unmatched token as a muted Badge
  - If selectedPlugins.length === 0 && unmatched.length === 0: render empty state (t.optimizer.emptyState + sample button using t.optimizer.sampleBtn)

Footer:
  - "분석하기" / "Analyze" Button
  - Disabled when selectedPlugins.length === 0
  - Disabled label: t.optimizer.analyzeBtnDisabled

```

1. **Create `components/PluginTypeInput.tsx`** — 'use client' component with autocomplete dropdown.

Props: `{ onSelect: (pluginId: string) => void; selectedIds: string[] }`

State:
- `query: string` — current input text
- `isOpen: boolean` — dropdown visibility
- `highlightedIndex: number` — keyboard navigation index (-1 = none)

Behavior:
- Use `filterPlugins(query, Object.values(PLUGINS))` from `lib/parse-mcp-list.ts` to get suggestions
- Filter out already-selected plugins (by checking against `selectedIds` prop)
- Render an Input (from shadcn/ui) with label `t.optimizer.typeLabel`, placeholder `t.optimizer.typePlaceholder`
- Below the input: absolute-positioned dropdown list showing filtered plugins
- Each suggestion row: plugin name + category + 1-line desc (truncated)
- Click on suggestion: call `onSelect(plugin.id)`, clear query, close dropdown

Keyboard navigation (onKeyDown on the Input):
- ArrowDown: increment highlightedIndex (wrap to 0)
- ArrowUp: decrement highlightedIndex (wrap to end)
- Enter: if highlightedIndex >= 0, select that suggestion
- Escape: close dropdown, reset highlightedIndex

Blur handling: close dropdown on blur (use a small setTimeout to allow click events on dropdown items to fire first)

CRITICAL Z-INDEX NOTE: The dropdown must use `z-50` and the parent container of input+dropdown must NOT have `overflow-hidden`. Use `relative` on the wrapper and `absolute left-0 right-0 top-full mt-1` on the dropdown. Add `overflow-y-auto max-h-64` to the dropdown for scrollability.

Accessibility:
- Input: `role="combobox"`, `aria-expanded={isOpen}`, `aria-autocomplete="list"`, `aria-activedescendant` pointing to highlighted item
- Dropdown: `role="listbox"`
- Each suggestion: `role="option"`, `aria-selected` when highlighted

Handle edge case: if `plugin.desc` is empty/undefined, show `t.optimizer.chipNoDesc` instead.
