## Gaps

### DATA-01: Plugin DB Count Below Minimum (9 vs 10-15)

**Requirement:** "주요 Plugin 10-15개가 DB에 추가되고, type: 'plugin'으로 분류된다"

**Actual state:** PLUGIN_FIELD_OVERRIDES has 9 `type: "plugin" as const` entries:
1. omc
2. superpowers
3. bkit-starter
4. bkit
5. taskmaster
6. gsd
7. fireauto
8. agency-agents
9. ralph

**Documentation discrepancy:** Phase 9 SUMMARY and VERIFICATION both claim 13 entries, including `repomix, context7, security, sentry, figma, playwright`. These 6 entries do NOT have `type: "plugin"` override in code — they receive `type: "mcp"` from DEFAULT_PLUGIN_FIELDS. Meanwhile, `taskmaster` and `gsd` have the override but were not in the SUMMARY's claimed list.

**Test alignment:** `plugins.test.ts` PLUGIN_TYPE_IDS correctly lists the 9 actual entries and tests pass.

**Gap severity:** Minor — 1 entry short of minimum. Adding 1 more `type: "plugin"` override closes the gap.

### Architectural Note

PLUGINS merge order `{ ...DEFAULT_PLUGIN_FIELDS, ...PLUGIN_FIELD_OVERRIDES[id], ...plugin }` is functionally correct but fragile: if a CORE_PLUGINS seed ever included a `type` key, it would silently override PLUGIN_FIELD_OVERRIDES. The PluginSeed type omits `type`, but TypeScript's structural typing doesn't enforce runtime absence. Not a current bug, noted as tech debt.
