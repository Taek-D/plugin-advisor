2. Create `lib/parse-mcp-list.ts` with four exported pure functions:

**`normalizeToken(raw: string): string`** — Lowercase, strip `(user):`, `(project):`, `@modelcontextprotocol/server-` prefix, `@scope/` prefix, `mcp-` prefix, `-mcp` suffix, non-alphanumeric except hyphen. Trim.

**`resolvePluginId(token: string, plugins: Plugin[]): string | null`** — Normalize the token, then try matching against each plugin's `id`, normalized `name`, and normalized `tag`. If no match, check a hardcoded alias map: `{ "brave": "brave-search", "github-mcp": "github" }`. Return plugin id or null.

**`parseMcpList(raw: string, pluginIds: string[]): ParseResult`** — Split into lines, strip blank/header lines. For each line: strip terminal control chars and Unicode symbols (checkmarks, x-marks), extract the first word-like token, normalize it, resolve against pluginIds. Bucket into matched vs unmatched. Deduplicate matched. Return `{ matched: string[], unmatched: string[] }`.

**`filterPlugins(query: string, plugins: Plugin[]): Plugin[]`** — If query length < 1, return []. Otherwise, case-insensitive substring match against `name`, `id`, and `tag` fields. Cap results at 8. This is the autocomplete data source.

Export the `ParseResult` type:
```typescript
export type ParseResult = {
  matched: string[];    // plugin ids resolved from DB
  unmatched: string[];  // raw tokens that didn't resolve
};
```

IMPORTANT: `parseMcpList` accepts `pluginIds: string[]` (not PLUGINS object) to keep it a pure function. The caller will pass `Object.keys(PLUGINS)`. Similarly, `resolvePluginId` accepts `plugins: Plugin[]` — the caller passes `Object.values(PLUGINS)`.

3. Create `lib/__tests__/parse-mcp-list.test.ts` with test cases covering all behaviors listed above. Use a small mock plugins array (3-4 plugins with realistic id/name/tag values like `{ id: "context7", name: "Context7", tag: "context7" }`) instead of importing the full PLUGINS object — keeps tests fast and isolated.
