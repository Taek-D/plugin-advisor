# Plan 12-01: i18n 업데이트 + 샘플 데이터 + 타입 뱃지

## Objective

/optimizer 페이지의 힌트/라벨을 MCP+Plugin 통합 문구로 업데이트하고, 샘플 데이터를 혼합 구성하며, 자동완성과 칩에 타입 뱃지를 추가한다.

## Tasks

<task id="12-01-01" title="i18n optimizer 키 값 업데이트 (ko.ts + en.ts)">
<context>
현재 optimizer 섹션의 pasteLabel, pasteHint, pastePlaceholder, emptyState가 "claude mcp list"만 안내한다.
MCP + Plugin 통합 문구로 변경해야 한다. Translations 타입은 수정하지 않는다 — 기존 키 값만 변경.
</context>
<steps>
1. `lib/i18n/ko.ts`의 optimizer 섹션에서 4개 키 값을 수정:
   - `pasteLabel`: "claude mcp list 결과" → "MCP / Plugin 목록"
   - `pasteHint`: "터미널에서 claude mcp list 명령어를 실행한 후 결과를 복사해서 붙여넣으세요" → "터미널에서 claude mcp list 또는 claude plugin list 명령어를 실행한 후 결과를 복사해서 붙여넣으세요"
   - `pastePlaceholder`: "claude mcp list 결과를 붙여넣으세요..." → "MCP / Plugin 목록을 붙여넣으세요..."
   - `emptyState`: "예: claude mcp list 결과를 붙여넣거나 플러그인을 검색하세요" → "MCP 또는 Plugin 목록을 붙여넣거나 플러그인을 검색하세요"

2. `lib/i18n/en.ts`의 optimizer 섹션에서 동일 4개 키 영문 값을 수정:
   - `pasteLabel`: "claude mcp list output" → "MCP / Plugin list"
   - `pasteHint`: "Run claude mcp list in your terminal, then copy and paste the output here" → "Run claude mcp list or claude plugin list in your terminal, then copy and paste the output here"
   - `pastePlaceholder`: "Paste your claude mcp list output here..." → "Paste your MCP / Plugin list here..."
   - `emptyState`: "Paste your claude mcp list output or search for plugins" → "Paste your MCP or Plugin list, or search for plugins"
</steps>

<automated>pnpm typecheck</automated>
</verify>
</task>

<task id="12-01-02" title="샘플 데이터를 MCP+Plugin 혼합으로 변경">
<context>
현재 handleSampleData는 MCP 3개만 포함한 텍스트 문자열을 parseMcpList에 넘긴다.
parseMcpList는 mixed format(MCP + Plugin)을 한 번에 파싱하면 isPluginList 플래그 충돌이 있다.
따라서 parseMcpList를 우회하고 ParseResult를 직접 생성하는 것이 가장 안정적이다.
</context>
<steps>
1. `components/OptimizerApp.tsx`에서 `handleSampleData` 수정:
   - 기존 코드: parseMcpList를 호출하여 텍스트 파싱
   - 새 코드: ParseResult를 직접 생성하여 5개 플러그인 ID 전달

```tsx
const handleSampleData = useCallback(() => {
  const sampleResult: ParseResult = {
    matched: ["context7", "playwright", "github", "superpowers", "omc"],
    unmatched: [],
  };
  handlePasteResult(sampleResult);
}, [handlePasteResult]);
```

2. 더 이상 사용하지 않는 import가 있으면 제거:
   - `parseMcpList`가 handleSampleData에서만 사용되었는지 확인
   - McpPasteInput에서도 사용하므로 import는 유지될 수 있음 — OptimizerApp에서 직접 import하지 않는 경우만 제거
</steps>
<verify>
<automated>pnpm typecheck</automated>
</verify>
</task>
